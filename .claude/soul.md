# Soul

This repository is the central place where the Freaks share common skills across all AgileFreaks projects.

It is a repository of knowledge — not just for software development, but for any skill or methodology the team finds valuable. If we've learned something worth repeating, it belongs here.

## Principles

**Distilled from practice, not theory.**
Every skill in this repo should come from real project experience. Don't add a skill because it sounds good in a blog post. Add it because it has been applied, tested, and found to work in the messy reality of actual projects.

**Quality over quantity.**
Each skill should be worth installing. A bloated library of mediocre skills is less valuable than a small collection of excellent ones. When in doubt, wait. A skill that's been proven in multiple contexts is more trustworthy than one that's been used once.

**Generic by design.**
Skills live here precisely because they are not tied to any one project. Project-specific configuration — paths, conventions, team processes — belongs in the consuming project's rules files. A skill that hardcodes project details is a skill that can't be shared.

**The humans decide what ships.**
Skills assist and accelerate. They don't replace judgment. Every skill should be honest about what it can verify automatically and what requires a human to confirm. Deferring honestly is a feature, not a limitation.

**Evolve deliberately.**
Versioning exists for a reason. When a skill improves, bump the version and document the change. When something is removed, explain why. Consuming projects should be able to trust that installing a version gives them stable, predictable behavior.
